# 用語集 — MQTT / Zig / ネットワーキング

本チュートリアルで登場する主要な用語をまとめたものです。

---

## MQTT 用語

| # | 用語 | 説明 |
|---|------|------|
| 1 | **Broker** | Publisher と Subscriber の間でメッセージを仲介するサーバー。トピックに基づいてメッセージをルーティングする。 |
| 2 | **Publisher** | トピックを指定してメッセージを送信するクライアント。 |
| 3 | **Subscriber** | トピックを購読し、そのトピックに発行されたメッセージを受信するクライアント。 |
| 4 | **Topic** | メッセージのルーティングに使われる UTF-8 文字列。`/` でレベルを区切る（例: `sensor/temperature/room1`）。 |
| 5 | **QoS 0 (At most once)** | メッセージを最大1回配信する。確認応答なし。最も軽量だが、メッセージが失われる可能性がある。 |
| 6 | **QoS 1 (At least once)** | メッセージを少なくとも1回配信する。PUBACK で確認する。重複配信の可能性がある。 |
| 7 | **QoS 2 (Exactly once)** | メッセージを正確に1回配信する。4段階のハンドシェイク（PUBLISH → PUBREC → PUBREL → PUBCOMP）を使用する。 |
| 8 | **Retain (保持メッセージ)** | Broker がトピックごとに最新の Retain メッセージを保存し、新しい Subscriber に即座に配信する仕組み。 |
| 9 | **Will (遺言メッセージ)** | クライアントが異常切断した際に Broker が自動的に発行するメッセージ。CONNECT パケットで設定する。 |
| 10 | **Clean Session** | `true` の場合、Broker は接続時に以前のセッション状態を破棄する。`false` の場合、セッション状態を維持する。 |
| 11 | **Packet ID (パケット識別子)** | QoS 1/2 のメッセージや SUBSCRIBE/UNSUBSCRIBE で使用される 16 ビットの識別子。 |
| 12 | **Keep Alive** | クライアントが一定間隔で PINGREQ を送信し、接続の生存を確認する仕組み。秒単位で指定する。 |
| 13 | **CONNECT** | クライアントが Broker への接続を開始するパケット。プロトコル名、バージョン、Client ID 等を含む。 |
| 14 | **CONNACK** | Broker が CONNECT に応答するパケット。接続の成否とセッション存在フラグを返す。 |
| 15 | **PUBLISH** | メッセージを発行するパケット。トピック名、QoS レベル、ペイロードを含む。 |
| 16 | **PUBACK** | QoS 1 の PUBLISH に対する確認応答パケット。 |
| 17 | **SUBSCRIBE** | トピックフィルタと QoS を指定して購読を要求するパケット。複数トピックを一度に購読できる。 |
| 18 | **SUBACK** | SUBSCRIBE に対する応答パケット。各トピックフィルタに対して許可された QoS またはエラーを返す。 |
| 19 | **UNSUBSCRIBE** | トピックの購読解除を要求するパケット。 |
| 20 | **UNSUBACK** | UNSUBSCRIBE に対する確認応答パケット。 |
| 21 | **PINGREQ** | クライアントから Broker への生存確認リクエスト。 |
| 22 | **PINGRESP** | Broker から PINGREQ に対する応答。 |
| 23 | **DISCONNECT** | クライアントが正常切断を通知するパケット。Will メッセージは発行されない。 |
| 24 | **Topic Wildcard `+`** | 単一レベルワイルドカード。トピックフィルタの1つのレベルに一致する（例: `sensor/+/temperature`）。 |
| 25 | **Topic Wildcard `#`** | 複数レベルワイルドカード。トピックフィルタの末尾でのみ使用でき、残りのすべてのレベルに一致する（例: `sensor/#`）。 |

## Zig 用語

| # | 用語 | 説明 |
|---|------|------|
| 26 | **Allocator** | メモリ割り当てを抽象化するインターフェース。`std.heap.page_allocator`、`GeneralPurposeAllocator` など複数の実装がある。 |
| 27 | **Slice (スライス)** | 配列の一部を指すポインタと長さのペア。`[]u8` は `u8` のスライス型。Zig で最も基本的なデータ参照方法。 |
| 28 | **Tagged Union (タグ付きユニオン)** | 複数の型のいずれか1つを保持できる型。`union(enum)` で定義し、`switch` で安全にパターンマッチする。MQTT パケットの表現に最適。 |
| 29 | **Comptime (コンパイル時実行)** | コンパイル時に評価される式やブロック。ジェネリクスの実現やコンパイル時の最適化に使用する。 |
| 30 | **Defer** | スコープ終了時に実行される式を登録する。リソース解放の保証に使用する（例: `defer allocator.free(buf)`）。 |
| 31 | **Errdefer** | エラーでスコープを抜ける場合のみ実行される Defer。正常パス時にはリソースを呼び出し元に引き渡し、エラーパスのみクリーンアップする。 |
| 32 | **Packed Struct** | フィールド間にパディングが入らない構造体。バイナリプロトコルのヘッダー解析に便利。`packed struct` で定義する。 |
| 33 | **ArrayList** | 動的に伸長可能な配列。Zig 0.16 では managed 版は `std.array_list.AlignedManaged(T, null)` を使用する。メッセージバッファの構築などに利用する。 |
| 34 | **HashMap** | キーと値のペアを格納するハッシュテーブル。`std.HashMap` や `std.StringHashMap` を使用する。セッション管理等に利用する。 |
| 35 | **Error Union (エラーユニオン)** | 値またはエラーを返す型。`!T` や `ErrorSet!T` と書く。Zig のエラーハンドリングの基盤。 |
| 36 | **Optional (オプショナル)** | 値が存在しない可能性を表す型。`?T` と書く。`null` チェックをコンパイラが強制する。 |
| 37 | **Sentinel-terminated (番兵終端)** | 配列やスライスの末尾に特定の値（番兵）を持つ型。`[:0]u8` はヌル終端文字列を表す。 |
| 38 | **@bitCast** | ビット列の再解釈を行うビルトイン関数。バイナリデータと構造体間の変換に使用する。 |
| 39 | **@intFromEnum** | enum 値を対応する整数値に変換するビルトイン関数。パケットタイプの数値化に使用する。 |

## Zig 0.16 std.Io 用語

| # | 用語 | 説明 |
|---|------|------|
| 40 | **std.Io** | Zig 0.16 で導入された統合 I/O フレームワーク。すべての I/O 操作は `io` パラメータを介して行われ、非同期・同期を透過的に切り替えられる。 |
| 41 | **std.Io.Threaded** | `std.Io` のスレッドベース実装。OS スレッドを使って並行処理を実現する。本チュートリアルの Broker で使用する主要な実装。 |
| 42 | **std.Io.Evented** | `std.Io` のイベントループベース実装。`epoll` / `kqueue` 等を使い、少数のスレッドで多数の接続を効率的に処理する。 |
| 43 | **std.Io.Mutex** | `std.Io` 対応のミューテックス。`.lockUncancelable(io)` でロックを取得し、`.unlock(io)` で解放する。`io` パラメータにより、ロック待機中も他のタスクを進行できる。 |
| 44 | **std.Io.RwLock** | `std.Io` 対応の読み書きロック。共有ロック（読み取り用）と排他ロック（書き込み用）を提供する。トピックツリーなど読み取り頻度の高いデータ構造に適する。 |
| 45 | **std.Io.Event** | `std.Io` 対応のイベント通知プリミティブ。あるタスクから別のタスクへシグナルを送る際に使用する。イベント駆動ブローカー設計の基盤。 |
| 46 | **std.Io.Queue(T)** | `std.Io` 対応のスレッドセーフキュー。型 `T` のメッセージをプロデューサーからコンシューマーへ安全に受け渡す。Broker 内部のメッセージパッシングに使用する。 |
| 47 | **io.async()** | `std.Io` の非同期タスク起動関数。関数を非同期に実行し、完了を待たずに次の処理に進む。 |
| 48 | **io.concurrent()** | `std.Io` の並行実行関数。複数の関数を並行して実行し、すべての完了を待つ。 |
| 49 | **std.Io.Group** | 複数の非同期タスクをグループ化し、まとめて管理・待機するための仕組み。タスクの一括キャンセルやすべての完了待ちに使用する。 |
| 50 | **std.process.Init (Juicy Main)** | Zig 0.16 の新しいエントリーポイントシグネチャ。`pub fn main(init: std.process.Init) !void` の形式で、`init.io()` から `std.Io` インスタンスを取得する。従来の `pub fn main() !void` を置き換える。 |
| 51 | **Cancelable error** | `std.Io` の操作がキャンセルされた際に返されるエラー。Graceful Shutdown 等でタスクの中断を通知するために使用する。 |
| 52 | **Future** | 非同期操作の結果を表す値。操作の完了を待つか、結果が既に利用可能かを確認できる。`std.Io` の非同期パターンの基本構成要素。 |
| 53 | **Backpressure (バックプレッシャー)** | コンシューマーの処理速度がプロデューサーより遅い場合に、プロデューサーの送信速度を制御する仕組み。`std.Io.Queue(T)` の容量制限やフロー制御と関連する。 |

## ネットワーキング用語

| # | 用語 | 説明 |
|---|------|------|
| 54 | **TCP (Transmission Control Protocol)** | 信頼性のある順序保証付きのバイトストリーム通信を提供するプロトコル。MQTT はデフォルトで TCP 上で動作する。 |
| 55 | **Socket (ソケット)** | ネットワーク通信のエンドポイント。IP アドレスとポート番号の組で識別される。 |
| 56 | **Stream (ストリーム)** | TCP 接続の読み書きに使用するインターフェース。Zig 0.16 では `reader.interface` / `writer.interface` フィールドでアクセスする。 |
| 57 | **Listener (リスナー)** | 指定ポートで着信接続を待ち受ける仕組み。Broker の入口。 |
| 58 | **Port (ポート)** | 0〜65535 の範囲の数値で、ホスト内のサービスを識別する。MQTT の標準ポートは **1883**（非暗号化）。 |
| 59 | **IP Address (IPアドレス)** | ネットワーク上のホストを識別する番号。IPv4 は 32 ビット（例: `127.0.0.1`）、IPv6 は 128 ビット。Zig 0.16 では `std.Io.net.IpAddress.parse()` で解析する。 |
| 60 | **Big Endian (ビッグエンディアン)** | 最上位バイトを先頭に配置するバイト順序。MQTT プロトコルではすべてのマルチバイト整数がビッグエンディアンで送信される。 |
| 61 | **Little Endian (リトルエンディアン)** | 最下位バイトを先頭に配置するバイト順序。x86/ARM などの CPU で一般的。ネットワーク送受信時に変換が必要。 |
| 62 | **Remaining Length (残余長)** | MQTT 固定ヘッダーの可変長エンコーディング。1〜4 バイトで最大 256 MB のパケットサイズを表現する。 |
| 63 | **Connection (コネクション)** | TCP の 3-way ハンドシェイクにより確立される双方向通信路。 |
| 64 | **Timeout (タイムアウト)** | 一定時間内に応答がない場合に接続やリクエストを打ち切る仕組み。Keep Alive と関連する。 |
| 65 | **localhost** | 自分自身を指す特別なホスト名。IP アドレス `127.0.0.1` に対応する。開発・テスト時に使用する。 |
